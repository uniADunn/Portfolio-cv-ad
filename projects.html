Honours Project

IoT-Based Dynamic Light Controller Guided by Historical Solar Irradiance

BSc (Hons) Software Development — Honours Project, Glasgow Caledonian University, 2025–2026

Overview

Indoor hydroponic systems typically use fixed lighting recipes that deliver the right quantity of light but not the natural daily patterns of real sunlight. This project addressed that gap by building and validating an IoT pipeline capable of reproducing historical solar diurnal spectral energy profiles indoors, targeting controlled environment agriculture (CEA) applications.

System Architecture

The system was designed as a modular IoT pipeline with four distinct layers. The Raspberry Pi 5 acted as the sensing endpoint, reading spectral data from an AS7341 11-channel sensor over I2C and publishing structured JSON payloads to a locally hosted Mosquitto MQTT broker. Communication between the Pi and a Windows laptop was secured through a Netbird encrypted peer-to-peer overlay, removing the need for port forwarding or static IPs. On the laptop side, a Python ingestion service subscribed to MQTT topics, validated incoming payloads, and wrote sensor readings and run metadata into a MySQL database. A scheduler managed run creation, solar target selection, and execution of simulation experiments. The Pi's sensor publishing script was deployed as a systemd service for automatic startup and restart on reboot.

Solar Target Generation

Hourly spectral energy targets were derived from three data sources. FAOSTAT crop yield data was used to identify high-performing crop-country-year combinations, selecting Belgium 2020 tomato data as the reference dataset. NASA POWER hourly solar irradiance data was retrieved for the country centroid coordinates at hourly resolution. ASTM G173-03 spectral fractions were then applied to split total hourly solar energy into band-level targets across blue (415–480nm), green (515–590nm), and red (630–680nm) wavelength bands.

Control Models

Three actuator simulation models were implemented and evaluated. The single coupled lamp model controlled all spectral bands with one duty value, meaning the band that reached its target first capped output for all others. The independent open-loop model assigned a separate duty value per band, calculated directly from the ratio of remaining energy to remaining capacity. The independent closed-loop model introduced tolerance-based feedback, adjusting duty values in discrete steps every 5 seconds by comparing predicted end-of-hour delivery against the target within a 5% tolerance band. All models were tested under both unscaled and scaled conditions, with results evaluated using Mean Absolute Error (MAE).

Results

The single coupled lamp failed under both scaled and unscaled conditions due to architectural coupling — performance did not improve with scaling, confirming the limitation was structural. Under unscaled conditions, the independent models tracked achievable bands accurately but hit hardware saturation on the red band. Under scaled conditions, the open-loop model achieved zero tracking error across all bands, and the closed-loop model maintained MAE within the 5% tolerance threshold across all three bands.

Stack

Python · Mosquitto MQTT · MySQL · Netbird · AS7341 Spectral Sensor · Raspberry Pi 5 · NASA POWER · ASTM G173 · FAOSTAT · Git / GitHub

Outcome

The engineering hypothesis was supported: the system can reproduce historical diurnal solar spectral energy profiles within defined hardware limits when independent per-band LED control is used and hardware output reaches or exceeds the maximum solar target. The system is ready to progress to biological validation, comparing dynamic solar-guided lighting against static recipes in a live hydroponic grow environment.
